# Структура проекта

Разработано: **Цветков Артём Денисович**, **СГТУ им. Гагарина Ю.А.**

Документ фиксирует актуальную структуру репозитория после отделения production-слоя от архивных материалов.

## Рабочие production-компоненты

```text
artifacts/
```

Сохранённая финальная модель, JSON-метаданные, метрики и пример входного запроса.

```text
ml_pipeline/
```

Feature engineering для production pipeline. Модуль используется при загрузке `artifacts/movie_revenue_pipeline.joblib`.

```text
scripts/
```

Воспроизводимые скрипты обучения и проверки финальной модели.

```text
backend/
```

FastAPI backend, который загружает готовый pipeline и отдаёт prediction API.

```text
frontend/
```

React + Vite frontend для пользовательской демонстрации веб-сервиса.

```text
notebooks/
```

Исследовательские ноутбуки для описания этапов анализа данных, обучения, интеграции и тестирования.

## Данные

```text
data/raw/
```

Исходный датасет до финальной очистки.

```text
data/processed/
```

Очищенный датасет `output.csv`, который используется для обучения production-модели.

## Документация

```text
docs/
```

Документация проекта, сценарии тестирования и материалы для магистерской работы.

```text
docs/assets/
```

Изображения и иллюстрации, используемые в документации.

## Архив

```text
archive/
```

Материалы ранних экспериментов и старых прототипов. Они отделены от рабочей архитектуры, чтобы production-код, документация и интерфейсы были понятнее.

```text
archive/legacy_apps/
```

Старые CLI/Streamlit-прототипы. Актуальный веб-сервис реализован через `backend/` и `frontend/`.

```text
archive/legacy_models/
```

Ранние исследовательские скрипты моделей.

```text
archive/legacy_helpers/
```

Вспомогательные скрипты анализа и визуализации.

```text
archive/legacy_datasets/
```

Архивные и промежуточные датасеты. Они не используются в финальной production-модели.

```text
archive/legacy_misc/
```

Дополнительные черновые материалы.
